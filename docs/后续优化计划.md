# PyFlow 后续优化计划

## 一、现有问题修复与完善

### 1.1 WH_MOUSE_LL 钩子 PyInstaller 兼容性

**问题**：`mouse_hook.py` 中 `SetWindowsHookExA` 在 PyInstaller 打包后返回错误 126（找不到模块），导致 Windows 下鼠标钩子方案不可用，回退到 pynput+回中方案。

**优化方案**：
- 尝试使用 `ctypes.windll.kernel32.GetModuleHandleW(None)` (宽字符版) 获取模块句柄
- 尝试将钩子回调注册在专用线程中并将 `hModule` 传 0（某些 Windows 版本允许）
- 研究 PyInstaller 的 `--uac-admin` 和运行时 DLL 搜索路径配置
- 如果以上方案均失败，在 PyInstaller 打包时将 `mouse_hook.py` 编译为独立 DLL 或使用 `ctypes.windll` 动态加载

**优先级**：高

### 1.2 ShowCursor 引用计数问题

**问题**：`system_utils.py` 中 `ShowCursor(False)` 只调用一次，Windows 的 `ShowCursor` 内部使用引用计数，多次调用隐藏后需要等量调用才能恢复。

**优化方案**：
- 维护一个显式的状态标志，而非依赖 `ShowCursor` 的引用计数
- 使用 `SetCursor` + 空白光标方案替代 `ShowCursor`：创建透明 `.cur` 文件，远程模式时 `SetCursor` 设为空白，返回时恢复
- 或者保存调用前的计数器值，恢复时循环调用 `ShowCursor(True)` 直到计数器恢复

**优先级**：高

### 1.3 剪贴板大文本处理

**问题**：`protocol.py` 中剪贴板消息的长度字段使用 `H`（unsigned short, 2字节），最大 65535 字节（~65KB），超出部分会被截断导致数据丢失。

**优化方案**：
- 将长度字段从 `H`（2字节）改为 `I`（4字节，unsigned int），支持最大 4GB 内容
- 或者实现分片传输：将大文本切分为多个消息片段，接收端拼接
- 添加对图片/文件剪贴板内容的基础支持（至少做到超大文本不崩溃）

**优先级**：中

## 二、性能优化

### 2.1 鼠标移动增量压缩与预测

**现状**：当前每帧都发送增量移动，网络波动时可能积压。

**优化方案**：
- 实现 `mouse_predictor.py` 中已预留的预测插值：客户端根据已收到的增量推算中间帧，减轻网络抖动感
- 服务端合并连续小幅增量（如 < 2px 的移动累积到 > 2px 再发送），减少消息频率
- 添加时间戳和序列号，支持客户端做平滑插值

### 2.2 键盘事件合并

**现状**：每次按键发送一个独立消息，快速输入时产生大量小包。

**优化方案**：
- 实现批量发送：短时间内（如 5ms）的连续键盘事件合并为一个消息包
- 使用 `TCP_NODELAY`（已启用）配合批量发送，兼顾延迟和效率

### 2.3 连接稳定性与心跳检测

**现状**：没有心跳机制，连接断开后需要依赖 TCP 超时或人工检测。

**优化方案**：
- 实现 `MSG_TYPE_HEARTBEAT`（协议已定义）的定时发送与检测
- 超过阈值时间未收到心跳则认为连接断开，触发重连或清理资源
- 心跳间隔建议 5 秒，超时阈值 15 秒

## 三、功能扩展

### 3.1 多客户端（多被控端）支持

**现状**：服务端只支持单客户端连接，`TCPServer` 的 `accept` 循环在第一次连接后不再接受新连接。

**优化方案**：
- 每个 `accept` 创建独立线程/协程处理
- 为每个客户端维护独立的切换状态和虚拟光标位置
- 支持配置多个远程屏幕，每个映射到不同被控端

**优先级**：高（KVM 类软件的核心功能）

### 3.2 TLS 加密通信

**现状**：所有数据明文传输，包括键鼠事件和剪贴板内容。

**优化方案**：
- 使用 Python `ssl` 模块在 TCP 连接上构建 TLS 层
- 支持自签名证书，配置文件中指定证书路径
- 提供 `--tls-cert` 和 `--tls-key` 命令行参数

**优先级**：中

### 3.3 拖拽文件传输

**现状**：剪贴板只支持文本同步。

**优化方案**：
- 扩展剪贴板同步，检测文件拖拽事件
- 实现文件传输协议：元数据（文件名、大小）+ 分片二进制传输
- 使用独立的文件传输通道，避免阻塞键鼠事件

**优先级**：低

### 3.4 GUI 配置界面

**现状**：只能通过命令行参数和 JSON 配置文件启动。

**优化方案**：
- 使用 Qt/Tkinter 开发轻量级配置 GUI
- 支持可视化屏幕布局拖拽配置
- 支持一键启动服务端/客户端
- 支持实时状态显示（连接状态、当前模式等）

**优先级**：低

### 3.5 macOS 和 Linux 适配

**现状**：Windows 下有 WH_MOUSE_LL 钩子方案，其他平台只有 pynput 回中回退方案。

**优化方案**：
- macOS：研究 `CGEventTap` 实现类似 Windows 钩子的底层拦截
- Linux：研究 `uinput`/`libei` 实现 Wayland 下的输入拦截
- 各平台分别实现 `PlatformMouseHook` 抽象类

**优先级**：中

## 四、代码质量

### 4.1 单元测试与集成测试

**现状**：`tests/` 目录存在但缺少测试用例。

**优化方案**：
- 为 `protocol.py` 添加打包/解包的单元测试
- 为 `coordinate_mapper.py` 添加坐标映射的单元测试
- 为 `event_dispatcher.py` 添加粘包处理的单元测试
- 为 `switch_controller.py` 添加状态机转换的单元测试
- 搭建本地双进程集成测试框架

### 4.2 日志系统完善

**现状**：日志主要使用 `print`，正式运行时难以追溯问题。

**优化方案**：
- 统一使用 `common/logger.py` 的 logging 系统
- 添加日志级别控制（DEBUG/INFO/WARNING/ERROR）
- 关键路径添加调试日志（切换事件、剪贴板同步、边缘检测触发）
- 日志输出到文件，方便用户反馈问题

### 4.3 错误处理与容错

**现状**：多处使用 `except Exception: pass` 吞掉了异常信息。

**优化方案**：
- 区分可预期异常和意外异常，意外异常至少记录到日志
- 网络断开时确保所有监听器和钩子正确清理
- 客户端重连成功后重新同步状态（剪贴板、屏幕尺寸等）

## 五、打包与分发

### 5.1 PyInstaller 打包优化

**现状**：`build.spec` 版本可能过旧，打包后钩子模块可能无法正常工作。

**优化方案**：
- 更新 `build.spec`，确保包含所有数据文件和隐式依赖
- 添加 `--hidden-import` 包含 `pynput`、`pyperclip` 等库的子模块
- 打包后进行端到端测试，验证功能完整性
- 支持单文件模式（`--onefile`）和目录模式（`--onedir`）

### 5.2 管理员权限自动提权

**现状**：Windows 下 WH_MOUSE_LL 钩子可能需要管理员权限。

**优化方案**：
- 程序启动时检测权限，如果不够则自动请求提权（UAC）
- 使用 `ctypes.windll.shell32.ShellExecuteW` 以管理员身份重新启动自身
