# PyFlow 打包指南

## 1 环境要求

- Python 3.8+
- PyInstaller 6.x
- 所有依赖库已安装（pynput, pyautogui 等）

## 2 已知问题及解决方案

### 问题1：pathlib 冲突

**错误信息**：

```
ERROR: The 'pathlib' package is an obsolete backport of a standard library package
```

**原因**：Anaconda 环境中有旧版 pathlib 包与 PyInstaller 冲突。

**解决方案**：

```bash
# 创建独立的虚拟环境
python -m venv venv_build

# 激活虚拟环境
.\venv_build\Scripts\activate

# 在虚拟环境中安装所有依赖
pip install -r requirements.txt
pip install pyinstaller pynput pyautogui pyperclip pygetwindow
```

### 问题2：pynput 打包失败

**错误信息**：

```
ModuleNotFoundError: No module named 'pynput'
```

**原因**：PyInstaller 默认无法自动识别 pynput 的子模块和隐藏导入。

**解决方案**：在 `build.spec` 中配置：

1. 使用 `collect_data_files` 和 `collect_submodules` 收集 pynput 数据
2. 手动添加所有 pynput 相关的 hiddenimports
3. 将虚拟环境的 site-packages 添加到 pathex

### 问题3：PyInstaller 使用错误的 Python

**原因**：PyInstaller 可能使用 Anaconda 的 Python 而非虚拟环境的 Python。

**解决方案**：在 `build.spec` 开头显式添加虚拟环境的 site-packages 到 sys.path。

## 3 build.spec 完整配置

```python
# -*- mode: python ; coding: utf-8 -*-
import os
import sys

# 关键：必须将虚拟环境的 site-packages 添加到路径
sys.path.insert(0, r'E:\LocalRepository\Project26.4\pyflow\venv_build\Lib\site-packages')

from PyInstaller.utils.hooks import collect_data_files, collect_submodules

block_cipher = None

# 收集 pynput 的数据和子模块
datas = collect_data_files('pynput')
subms = collect_submodules('pynput')

# 手动添加所有需要的隐藏导入
hiddenimports = subms + [
    'pynput',
    'pynput.mouse',
    'pynput.keyboard',
    'pynput._info',
    'pynput._util',
    'pynput._util.win32',
    'pynput.mouse._base',
    'pynput.mouse._win32',
    'pynput.keyboard._base',
    'pynput.keyboard._win32',
    'pygetwindow',
    'pyautogui',
    'pyperclip',
]

a = Analysis(
    ['main.py'],
    pathex=[r'E:\LocalRepository\Project26.4\pyflow\venv_build\Lib\site-packages'],
    binaries=[],
    datas=datas,
    hiddenimports=hiddenimports,
    hookspath=[],
    hooksconfig={},
    runtime_hooks=[],
    excludes=[],
    win_no_prefer_redirects=False,
    win_private_assemblies=False,
    cipher=block_cipher,
    noarchive=False,
)

pyz = PYZ(a.pure, a.zipped_data, cipher=block_cipher)

exe = EXE(
    pyz,
    a.scripts,
    [],
    exclude_binaries=True,
    name='PyFlow',
    debug=False,
    bootloader_ignore_signals=False,
    strip=False,
    upx=True,
    console=True,
    disable_windowed_traceback=False,
    argv_emulation=False,
    target_arch=None,
    codesign_identity=None,
    entitlements_file=None,
    icon=None,
)

coll = COLLECT(
    exe,
    a.binaries,
    a.zipfiles,
    a.datas,
    strip=False,
    upx=True,
    upx_exclude=[],
    name='PyFlow_v4',
)
```

## 4 打包步骤

### 4.1 准备工作

```bash
# 1. 创建虚拟环境
python -m venv venv_build

# 2. 激活虚拟环境
.\venv_build\Scripts\activate

# 3. 安装依赖
pip install -r requirements.txt
pip install pyinstaller pynput pyautogui pyperclip pygetwindow

# 4. 验证 pynput 安装成功
python -c "import pynput; print(pynput.__file__)"
```

### 4.2 执行打包

```bash
# 方式1：使用虚拟环境中的 Python 模块方式
.\venv_build\Scripts\python.exe -m PyInstaller build.spec --clean

# 方式2：直接使用虚拟环境中的 pyinstaller
.\venv_build\Scripts\pyinstaller.exe build.spec --clean
```

### 4.3 验证打包结果

```bash
# 测试帮助信息
.\dist\PyFlow_v4\PyFlow.exe --help

# 应该看到正常输出的帮助信息
```

## 5 使用打包后的程序

### 5.1 文件夹结构

```
dist\PyFlow_v4\
├── PyFlow.exe          # 主程序
└── _internal\          # 依赖库和Python解释器
```

### 5.2 运行方式

**服务端（操控端电脑）**：

```bash
PyFlow.exe --server --ip 0.0.0.0 --port 12345
```

**客户端（被控端电脑）**：

```bash
PyFlow.exe --client --ip <服务端IP> --port 12345
```

### 5.3 权限要求

| 模式   | 权限要求       | 原因                      |
| ------ | -------------- | ------------------------- |
| 服务端 | 建议管理员     | 避免网络权限问题          |
| 客户端 | **必须管理员** | pynput 需要高权限模拟输入 |

### 5.4 常见错误

| 错误信息                                 | 解决方案                           |
| ---------------------------------------- | ---------------------------------- |
| `No module named 'pynput'`               | 重新打包，确保 build.spec 正确配置 |
| `pynput._util.win32.NotImplementedError` | 以管理员权限运行                   |
| 连接失败                                 | 检查防火墙和 IP 地址               |

## 6 故障排查

### 6.1 打包时目录被占用

如果遇到 `PermissionError: [WinError 32]`：

```bash
# 关闭所有运行的 PyFlow.exe 进程
Get-Process PyFlow -ErrorAction SilentlyContinue | Stop-Process -Force

# 或使用新的输出目录名（修改 build.spec 中的 name 参数）
```

### 6.2 虚拟环境问题

如果打包后运行报错，检查：

```bash
# 确认虚拟环境中的 Python 能导入 pynput
.\venv_build\Scripts\python.exe -c "import pynput; print('OK')"

# 确认 PyInstaller 是虚拟环境中的版本
.\venv_build\Scripts\pyinstaller.exe --version
```
