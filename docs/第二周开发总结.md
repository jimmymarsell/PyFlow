# PyFlow 第2周开发总结

## 项目概述

**PyFlow** 是基于 Python 开发的跨终端键盘鼠标控制工具（Python 版 Deskflow），采用 C/S 架构，实现局域网内键鼠共享。

### 项目定位

- 学习性质项目，代码模块化、易读
- 对标 Deskflow 官方功能
- 轻量化版本（非工业级）

## 架构设计（已更新）

### 角色定义

| 角色                 | 运行位置                         | 说明                           |
| -------------------- | -------------------------------- | ------------------------------ |
| **Server（主控端）** | 用户的电脑（鼠标键盘连接的电脑） | 监听本地输入，发送到被控端     |
| **Client（被控端）** | 其他电脑                         | 接收主控端事件，在本地执行操作 |

### 模块关系

```
                        继承关系
                        ==========
TCPServer <|-- ShareServer
TCPClient <|-- ShareClient

                        TCP连接
主控端 (server/)      ──────────────►  被控端 (client/)
  │                                        │
  ├── mouse_listener.py                   ├── mouse_controller.py
  ├── keyboard_listener.py (第三周)        ├── keyboard_controller.py (第三周)
  └── clipboard_monitor.py (第三周)       └── clipboard_sync.py (第三周)
```

### 事件流程

**主控端（Server）**：

1. MouseListener 监听本地鼠标事件
2. KeyboardListener 监听本地键盘事件（第三周）
3. ClipboardMonitor 监控本地剪贴板（第三周）
4. 事件通过 pack\_\* 函数序列化为二进制
5. 通过 TCP 发送到被控端

**被控端（Client）**：

1. 接收主控端发送的二进制数据
2. EventDispatcher 根据消息类型分发
3. MouseController/KeyboardController 调用 pynput 执行操作
4. ClipboardSync 更新本地剪贴板
5. MousePredictor 进行本地预测和插值，平滑鼠标移动（第二周优化）

## 核心优化特性

### 1. 本地预测与插值

为了改善高延迟网络下的鼠标卡顿问题，被控端实现了本地预测算法：

```
网络延迟: ~100ms

无优化: 鼠标每100ms跳跃一次，明显卡顿
有优化: 鼠标平滑移动到目标位置，感觉流畅
```

**MousePredictor 类**：

- 记录位置历史，计算速度向量
- 支持预测未来位置
- 支持平滑插值

**插值线程**：

- 以 60Hz 频率平滑移动鼠标
- 每次移动距离的 30%，减少过快移动

### 2. 分辨率缩放

支持不同分辨率屏幕间的坐标转换：

```python
class ScreenConfig:
    def scale_x(self, x, target_width):
        """按比例缩放X坐标"""

    def scale_y(self, y, target_height):
        """按比例缩放Y坐标"""
```

**示例**：

```
主控端屏幕: 2560 × 1600
被控端屏幕: 1920 × 1080

主控端鼠标 x=1280（屏幕中央）
被控端鼠标 x=960（屏幕中央）
x' = 1280 * (1920/2560) = 960
```

## 核心协议设计

### 消息类型

| 类型                  | 值   | 说明               |
| --------------------- | ---- | ------------------ |
| MSG_TYPE_MOUSE_MOVE   | 0x01 | 鼠标移动事件       |
| MSG_TYPE_MOUSE_BUTTON | 0x02 | 鼠标按钮事件       |
| MSG_TYPE_MOUSE_SCROLL | 0x03 | 鼠标滚动事件       |
| MSG_TYPE_KEYBOARD     | 0x04 | 键盘事件（预留）   |
| MSG_TYPE_CLIPBOARD    | 0x05 | 剪贴板事件（预留） |
| MSG_TYPE_HEARTBEAT    | 0xFF | 心跳包             |

### 协议格式

#### 鼠标移动事件 (9字节)

```
| 1字节类型 | 4字节X | 4字节Y |
| 0x01      | int32 | int32 |
```

#### 鼠标按钮事件 (13字节)

```
| 1字节类型 | 1字节按钮 | 1字节状态 | 4字节X | 4字节Y | 1字节保留 |
| 0x02      | button   | pressed  | int32  | int32  | 0        |
```

#### 鼠标滚动事件 (13字节)

```
| 1字节类型 | 4字节X | 4字节Y | 4字节dx | 4字节dy |
| 0x03      | int32  | int32  | int32   | int32   |
```

### 打包/解包示例

```python
# 打包鼠标移动
data = pack_mouse_move(100, 200)  # 返回 9 字节

# 解包鼠标移动
```

### 4. 事件分发模式

```python
class EventDispatcher:
    def __init__(self):
        """
        初始化事件分发器
        """
        self._handlers = {}
        self._buffer = b""

    def register(self, msg_type, handler):
        """
        注册消息类型对应的处理器
        :param msg_type: 消息类型
        :param handler: 处理函数
        """
        self._handlers[msg_type] = handler

    def dispatch(self, data):
        """
        分发事件到对应的处理器
        关键步骤：
        1. 将数据添加到缓冲区
        2. 循环解析完整消息
        3. 调用处理器执行
        """
        self._buffer += data
        while len(self._buffer) > 0:
            msg_type = get_msg_type(self._buffer)
            if msg_type is None:
                break
            expected_size = get_msg_size(msg_type)
            if expected_size == 0:
                self._buffer = b""
                break
            if len(self._buffer) < expected_size:
                break
            msg_data = self._buffer[:expected_size]
            self._buffer = self._buffer[expected_size:]
            handler = self._handlers.get(msg_type)
            if handler:
                handler(msg_data)
```

### 5. 鼠标移动插值

```python
class ShareClient:
    def _interpolation_loop(self):
        """
        插值循环
        以60Hz的频率平滑移动鼠标到目标位置
        """
        while not self.stop_interpolation.is_set():
            dx = self.target_x - self.current_x
            dy = self.target_y - self.current_y

            step_x = dx * 0.3  # 每次移动30%距离
            step_y = dy * 0.3

            new_x = int(self.current_x + step_x)
            new_y = int(self.current_y + step_y)

            self.mouse_controller.move_to(new_x, new_y)
            self.current_x = new_x
            self.current_y = new_y

            time.sleep(0.016)  # ~60Hz
```

## 使用方法

### 启动主控端（你的电脑）

```bash
PyFlow.exe --server --port 12345
```

主控端启动后等待被控端连接。

### 启动被控端（另一台电脑）

```bash
PyFlow.exe --client --ip <主控端IP> --port 12345
```

### 工作原理

1. 主控端运行在"你的电脑"上（鼠标键盘连接的电脑）
2. 被控端运行在"其他电脑"上
3. 主控端监听本地鼠标、键盘事件并发送到被控端
4. 被控端执行相应的鼠标、键盘操作

### 使用场景

```
┌─────────────────┐                      ┌─────────────────┐
│  主控端 Server   │  ───发送事件 ───►    │  被控端 Client   │
│                  │                      │                  │
│ • 监听本地鼠标     │                      │ • 接收事件        │
│ • 监听本地键盘     │                      │ • 执行鼠标操作     │
│ • 监控剪贴板      │                      │ • 执行键盘操作     │
│ • 你的电脑        │                      │ • 更新剪贴板      │
│                  │                      │ • 另一台电脑      │
└─────────────────┘                      └─────────────────┘
```

## 遇到的问题及解决方案

### 问题1：pynput 打包格式不一致

**现象**：鼠标事件打包后长度不符合预期

**原因**：struct 格式符使用不当

**解决方案**：

```python
# 使用正确的大端序格式
STRUCT_FORMAT = ">Biif"  # 1字节 + 4字节 + 4字节
```

### 问题2：鼠标按钮值转换

**现象**：按钮事件中 button 值传递后无法正确识别

**原因**：pynput.mouse.Button 枚举值需要转换

**解决方案**：

```python
# 按钮名称映射
BUTTON_MAP = {
    'left': 1,
    'right': 2,
    'middle': 3,
    'x1': 4,
    'x2': 5,
}

# 发送时使用按钮名称
button.name  # 'left', 'right' 等
```

### 问题3：TCP 粘包/拆包

**现象**：解包时报 "unpack requires a buffer of X bytes"

**原因**：TCP 不保证数据边界的完整性

**解决方案**：在 EventDispatcher 中实现缓冲区，按消息长度拆包

### 问题4：网络延迟导致鼠标卡顿

**现象**：高延迟网络下鼠标移动卡顿

**原因**：鼠标位置每100ms跳跃一次

**解决方案**：

1. 被控端实现插值线程
2. 以 60Hz 频率平滑移动鼠标到目标位置
3. 每次移动距离的 30%

### 问题5：防火墙阻止 ping

**现象**：A 能 ping 通 B，但 B ping 不通 A

**原因**：Windows 防火墙默认阻止入站 ICMP

**解决方案**：

```powershell
netsh advfirewall firewall add rule name="ICMP Echo" dir=in action=allow protocol=icmpv4:8,any
```

## 已知局限

### 1. 仅支持鼠标事件

- 当前实现了鼠标移动、点击、滚动
- 键盘事件和剪贴板同步尚未实现（第3周任务）

### 2. 无屏幕边界检测

- 当前没有实现屏幕边界检测
- 无法自动切换到其他被控电脑
- 需要手动移动鼠标到边界

### 3. 无心跳检测

- 当前没有实现心跳包机制
- 断开连接后需手动重启

### 4. 无自动重连

- 客户端断开后不会自动重连
- 需手动重启客户端
