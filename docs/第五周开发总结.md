## 核心Bug修复详情

### Bug 1：远程模式下所有事件无条件转发

**问题**：原 `share_server.py` 中，无论 NORMAL 还是 REMOTE 模式，所有鼠标/键盘事件都无条件转发到被控端，导致两台电脑同时响应。

**修复**：
- NORMAL 模式：不发送任何事件到被控端，仅做边缘检测
- REMOTE 模式：发送增量移动（dx/dy），按钮和滚动坐标设为 (0,0)

### Bug 2：边缘弹跳

**问题**：光标出现在远程屏幕边缘后立即触发返回检测，导致来回切换（infinitely bouncing）。

**修复**：
- `coordinate_mapper.py` 中 EDGE_OFFSET=15：光标不放在边缘像素，向内偏移 15px
- `share_server.py` 中 `_switch_grace_until`：切换后 500ms 冷却时间
- 返回检测从客户端移到服务端：服务端追踪虚拟光标位置

### Bug 3：边缘检测轮询延迟

**问题**：原 `screen_edge.py` 使用 `pyautogui.position()` 每 50ms 轮询，延迟大。

**修复**：改用 pynput `on_move` 回调，鼠标事件直接触发检测。

## 新增协议

### MSG_TYPE_MOUSE_DELTA (0x08)

```python
STRUCT_FORMAT_DELTA = ">Bhh"  # 1字节类型 + 2字节dx(int16) + 2字节dy(int16)
# 总计 5 字节
```

用于远程模式下发送鼠标移动增量，取代原来的绝对坐标移动。

### SWITCH 消息扩展

```python
STRUCT_FORMAT_SWITCH = ">BiiiHH"  # 增加 screen_w, screen_h
# 总计 17 字节
```

被控端根据 screen_w/screen_h 进行坐标缩放，支持不同分辨率。

## 切换状态机

```
NORMAL ──边缘命中──→ SWITCHING ──立即──→ REMOTE
   ↑                                         │
   └─────────返回本地坐标──────────────────────┘
                   (虚拟光标到达对向边缘)
```

- **NORMAL**：本地控制，只做边缘检测
- **SWITCHING**：瞬时过渡态，切换后立即进入 REMOTE
- **REMOTE**：远程控制，隐藏光标 + 发送增量
