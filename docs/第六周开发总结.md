## 核心改动详情

### 1. 键盘远程模式抑制

**问题**：远程模式下，按键同时发送到被控端和本地操作系统。例如按 Ctrl+C，本地和被控端都会复制。

**方案**：利用 pynput `keyboard.Listener` 的 `suppress` 参数：

```python
class KeyboardListener:
    def set_suppress(self, suppress):
        """切换 suppress 模式，需要重建 listener"""
        if self._suppress != suppress:
            self._suppress = suppress
            if self.listener:
                self.listener.stop()
                self.listener = None
            self._start_listener()
```

- 切换到远程：`keyboard_listener.set_suppress(True)` → 所有按键被拦截，不传达到 OS
- 返回本地：`keyboard_listener.set_suppress(False)` → 恢复正常输入

**联动点**：
- `_on_switch_to_remote` → `set_suppress(True)`
- `_on_switch_to_local` → `set_suppress(False)`
- `_on_switch_state_change(NORMAL)` → `set_suppress(False)`
- `_handle_client` 断开时 → `set_suppress(False)` 安全恢复

### 2. 剪贴板协议 Bug 修复

**问题**：`protocol.py` 中 `get_msg_size(MSG_TYPE_CLIPBOARD)` 返回 3，但剪贴板是变长消息（头3字节 + 内容），导致 dispatcher 只读 3 字节就认为消息完整，内容全部丢失。

**修复**：

```python
# 修复前
elif msg_type == MSG_TYPE_CLIPBOARD:
    return 3  # ← 错误！只读了头部，内容丢失

# 修复后
elif msg_type == MSG_TYPE_CLIPBOARD:
    return 0  # ← 返回 0 触发变长消息解析逻辑
```

`event_dispatcher.py` 中已有的变长消息处理分支：
```python
if expected_size == 0:
    if msg_type == MSG_TYPE_CLIPBOARD:
        if len(self._buffer) < 3:
            break
        _, length = struct.unpack(">BH", self._buffer[:3])
        expected_size = 3 + length  # 头部 + 内容
```

### 3. 剪贴板双向同步

**之前**：只有 服务端→客户端 单向同步。客户端没有 ClipboardMonitor。

**之后**：双向同步，两端各自监控本地剪贴板变化。

```
主控端复制 → ClipboardMonitor 检测变化 → 发送到被控端 → ClipboardSync 写入剪贴板
被控端复制 → ClipboardMonitor 检测变化 → 发送到主控端 → pyperclip.copy() 写入剪贴板
```

**ShareServer 改动**：
- `_on_clipboard_change`：发送前检查 `_last_remote_clipboard`，如果与当前内容相同则跳过（防回环）
- 新增 `_on_clipboard_from_client`：接收来自被控端的剪贴板内容，写入本地 pyperclip，更新 `_last_remote_clipboard` 和 `clipboard_monitor.set_last_content()`
- `_setup_dispatcher`：注册 `MSG_TYPE_CLIPBOARD` 处理器

**ShareClient 改动**：
- 新增 `ClipboardMonitor`：连接后启动，监控本地剪贴板变化
- `_on_local_clipboard_change`：检测到变化后发送到主控端，跳过 `_last_remote_clipboard`（防回环）
- `_on_clipboard`：接收来自主控端的剪贴板内容，更新 `_last_remote_clipboard`、`clipboard_sync` 和 `clipboard_monitor.set_last_content()`
- `disconnect`：停止 ClipboardMonitor

### 4. 防回环机制

**问题**：A 复制 → 发到 B → B 写入剪贴板 → B 的 ClipboardMonitor 检测到变化 → 又发回 A → 无限循环。

**方案**：三层防护：

| 层 | 机制 | 位置 |
|---|------|------|
| 1 | `_last_remote_clipboard` 比对 | 服务端/客户端 `_on_clipboard_change` |
| 2 | `set_last_content()` 同步更新 | ClipboardMonitor / ClipboardSync |
| 3 | 收到远端内容后立即更新 `_last_content` | 防止轮询检测到"新"变化 |
