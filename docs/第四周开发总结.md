# PyFlow 第4周开发总结

## 项目概述

**PyFlow** 是基于 Python 开发的跨终端键盘鼠标控制工具（Python 版 Deskflow），采用 C/S 架构，实现局域网内键鼠共享。

## 功能说明

### 1. ScreenLayout 数据类 (server/screen_layout/screen_layout.py)

| 属性 | 类型 | 说明 |
| --- | --- | --- |
| screen_id | str | 屏幕唯一标识符 |
| host | str | 被控端 IP 地址 |
| port | int | 被控端端口 |
| width | int | 屏幕宽度（像素） |
| height | int | 屏幕高度（像素） |
| direction | str | 拼接方向 (left/right/top/bottom) |
| alignment | str | 对齐方式 (left/right/top/bottom) |
| offset_x | int | X方向微调偏移 |
| offset_y | int | Y方向微调偏移 |
| edge_margin | int | 边缘触发阈值 |
| is_local | bool | 是否为本地屏幕 |

### 2. LayoutManager 布局管理器 (server/screen_layout/layout_manager.py)

| 方法 | 说明 |
| --- | --- |
| `add_screen(screen)` | 添加屏幕到布局 |
| `get_screen(screen_id)` | 根据 ID 获取屏幕 |
| `get_local_screen()` | 获取本地屏幕 |
| `get_remote_screens()` | 获取所有远程屏幕 |
| `get_screen_at(x, y)` | 根据坐标查找对应屏幕 |
| `get_adjacent_screen(from_screen, direction)` | 获取指定方向相邻的屏幕 |
| `load_config(config_path)` | 从文件加载布局配置 |
| `save_config(config_path)` | 保存布局配置到文件 |

### 3. EdgeDetector 边界检测器 (server/screen_layout/edge_detector.py)

| 方法 | 说明 |
| --- | --- |
| `is_at_edge(x, y)` | 检测鼠标是否在切换边界上 |
| `get_edge_direction(x, y)` | 获取鼠标所在的边界方向 |
| `check(x, y)` | 检查鼠标位置，决定是否触发切换 |
| `is_moving_toward_edge(x, y, prev_x, prev_y)` | 检测鼠标是否正在向边界方向移动 |
| `get_edge_position()` | 获取边界位置坐标 |

### 4. CoordinateMapper 坐标映射器 (server/screen_layout/coordinate_mapper.py)

| 方法 | 说明 |
| --- | --- |
| `map_to_remote(from_screen, to_screen, x, y)` | 将坐标从源屏幕映射到目标（远程）屏幕 |
| `map_to_local(from_screen, to_screen, x, y)` | 将坐标从远程屏幕映射回本地屏幕 |
| `calculate_ratio(from_screen, to_screen)` | 计算两个屏幕之间的宽高比例 |

支持的映射方向：
- 左对齐 (left): 源屏幕右边界 → 目标屏幕左边界
- 右对齐 (right): 源屏幕左边界 → 目标屏幕右边界
- 上对齐 (top): 源屏幕下边界 → 目标屏幕上边界
- 下对齐 (bottom): 源屏幕上边界 → 目标屏幕下边界

### 5. SwitchController 切换控制器 (server/screen_layout/switch_controller.py)

| 方法/属性 | 说明 |
| --- | --- |
| `state` | 当前状态 (NORMAL/SWITCHING/REMOTE/RETURNING) |
| `on_mouse_move(x, y)` | 处理鼠标移动事件 |
| `switch_to_remote(target_screen, x, y)` | 切换到远程屏幕 |
| `return_to_local(x, y)` | 返回本地屏幕 |
| `is_remote_control()` | 是否正在控制远程屏幕 |
| `reset()` | 重置控制器到初始状态 |

**状态转换图：**
```
NORMAL → SWITCHING (鼠标到达边界)
SWITCHING → REMOTE (切换完成)
REMOTE → RETURNING (鼠标到达远程边界)
RETURNING → NORMAL (返回完成)
```

## 协议扩展

### 新增消息类型

```python
MSG_TYPE_SWITCH = 0x06        # 切换屏幕命令
MSG_TYPE_SWITCH_ACK = 0x07    # 切换确认
```

### 切换消息格式

```
Switch 消息格式：
struct.pack('>Biii', MSG_TYPE_SWITCH, target_x, target_y, action)

参数说明：
- target_x: 目标X坐标
- target_y: 目标Y坐标
- action: 切换动作 (0=切换到远程, 1=返回本地)
```

## 架构集成

### ShareClient 新增功能

```python
def _on_switch(self, target_x, target_y, action):
    """
    处理屏幕切换命令
    """
    if action == 0:
        self._stop_interpolation()
        self.target_x = target_x
        self.target_y = target_y
        self.current_x = target_x
        self.current_y = target_y
        self.mouse_controller.move_to(target_x, target_y)
```

## 技术要点

1. **状态机设计**：使用枚举定义状态转换，保证切换逻辑清晰
2. **坐标映射算法**：基于比例的坐标转换，支持不同分辨率屏幕
3. **边界检测**：可配置的边缘触发阈值，防止误触发
4. **模块化设计**：各组件职责分离，便于维护和测试
