# PyFlow 鼠标切换优化分析与修改方案

## 1. 问题概述

当前键盘和剪贴板功能正常，但鼠标在不同电脑之间的切换不够丝滑，无法达到物理多屏拼接的体验。核心表现为：
- 切换到远程屏幕时鼠标跳跃而非平滑穿越
- 远程控制时鼠标移动有延迟和卡顿
- 切换回本地时鼠标位置丢失或不对
- 两个鼠标光标同时可见
- 边缘检测响应慢，可能误触发或不触发

---

## 2. 问题根因分析

按严重程度排序，从最影响体验的问题开始：

### 2.1 【严重】边缘检测使用轮询而非事件驱动

**文件**: `server/screen_edge.py`

```python
def _monitor_loop(self):
    while not self._stop_event.is_set():
        try:
            x, y = pyautogui.position()  # 每50ms轮询一次
            self._check_edges(x, y)
        except Exception:
            pass
        self._stop_event.wait(0.05)       # 50ms延迟
```

**问题**:
- 轮询间隔 50ms，最坏情况下有 50ms 的边缘检测延迟
- `pyautogui.position()` 本身有系统调用开销，在 Windows 上可能更慢
- pynput 的 `MouseListener` 已经在另一个线程实时监听鼠标移动，但 `ScreenEdge` 完全忽略了这些事件
- **两套独立的鼠标位置获取机制**（pynput 回调 vs pyautogui 轮询），互相不协调

**影响**: 切换响应延迟 50ms+，鼠标快移时可能直接越过边缘阈值区而未被检测到。

---

### 2.2 【严重】鼠标移动使用绝对坐标传输而非增量传输

**文件**: `server/mouse_listener.py`, `common/protocol.py`

```python
def _on_move(self, x, y):
    self._last_x = x
    self._last_y = y
    if self.on_mouse_event:
        data = pack_mouse_move(x, y)    # 发送绝对坐标
        self.on_mouse_event(data)
```

**问题**:
- 每次鼠标移动发送绝对坐标 (9字节/事件)
- pynput 的鼠标移动事件频率极高，Windows 上可达 125-1000Hz（取决于鼠标回报率）
- 高频率的绝对坐标传输导致大量网络包，每个包都需要完整的坐标信息
- 在远程屏幕上，绝对坐标被直接使用或做插值，无法还原连续的移动轨迹

**对比物理多屏**: 操作系统在多屏之间使用增量坐标（delta X, delta Y），光标连续移动，无跳跃。

---

### 2.3 【严重】客户端插值实现存在根本性缺陷

**文件**: `client/share_client.py`

```python
def _interpolation_loop(self):
    while not self._stop_interpolation.is_set():
        dx = self.target_x - self.current_x
        dy = self.target_y - self.current_y
        distance = math.sqrt(dx * dx + dy * dy)

        if distance < 1:
            time.sleep(0.001)
            continue

        step_x = dx * 0.3    # 固定0.3的指数平滑
        step_y = dy * 0.3
        # ...
        time.sleep(0.016)     # ~60Hz
```

**问题**:
- **平滑系数 0.3 造成人为延迟**: 光标永远无法一步到达目标位置，始终有滞后
- **点击时中断插值**: `_on_mouse_button` 调用 `_stop_interpolation()`，然后直接跳到绝对位置，造成视觉跳跃
- **线程管理不安全**: `_start_interpolation` 只在 `interpolation_thread is None` 时创建新线程，但线程结束后没有重置为 None
- **睡眠不精确**: `time.sleep(0.016)` 在 Windows 上精度约 15ms，实际帧率不稳定
- **与 MousePredictor 重复**: 已创建了 `MousePredictor` 但从未在插值中使用

---

### 2.4 【严重】远程控制时本地鼠标光标未隐藏

**文件**: `server/share_server.py`

```python
def _on_mouse_event(self, data):
    if self.switch_controller and self.switch_controller.is_remote_control():
        return                          # 只是跳过发送，本地光标还在动！
```

**问题**:
- 切换到远程控制后，本地鼠标事件只是被丢弃（return），但**本地光标仍然可见且响应鼠标移动**
- 用户看到两个光标：本地光标随物理鼠标移动，远程光标通过网络延迟移动
- 切回本地时，本地光标可能已经不在预期位置

**对比物理多屏**: 只有一个光标，光标在不同屏幕间无缝移动。

---

### 2.5 【严重】坐标映射逻辑有误

**文件**: `server/screen_layout/coordinate_mapper.py`

各映射函数的逻辑问题：

**`_map_from_left` (右方向切换)**:
- 当 alignment 为 `right` 时: `new_x = to_screen.width - 1`，即光标出现在远程屏幕右边缘
- **这是错误的**: 如果鼠标从本地屏幕右边缘移向右边，应该出现在远程屏幕的左边缘，而不是右边缘
- 只有 `alignment='left'` 时行为正确（光标出现在左边缘 x=0）

**`_map_from_right` (左方向切换)** 类似问题

**`_map_from_top` 和 `_map_from_bottom`** 也有类似错误

**正确的映射应该是什么？**:
- 鼠标从本地右边缘移出 → 出现在远程屏幕左边缘，Y 坐标按比例映射
- 鼠标从本地左边缘移出 → 出现在远程屏幕右边缘，Y 坐标按比例映射
- 进入点的 X 坐标应该是固定值（到达边缘），Y 坐标按比例映射，而不是将 X 坐标也做比例缩放

---

### 2.6 【中等】没有 TCP_NODELAY，网络延迟叠加

**文件**: `network/tcp_server.py`, `network/tcp_client.py`

- TCP 默认启用 Nagle 算法，会将小包合并发送
- 鼠标移动包只有 9 字节，Nagle 会延迟最多 200ms 再发送
- 对于实时性要求极高的鼠标控制，这会显著增加延迟感知

---

### 2.7 【中等】没有鼠标事件节流/采样率控制

**文件**: `server/mouse_listener.py`

- pynput 的 `on_move` 回调频率等于鼠标回报率（125-1000Hz）
- 每次回调都立即发送一个网络包
- 在高回报率鼠标上，1秒可能发送 1000 个 9 字节的鼠标移动包
- 大量小包会增加网络负担和接收端处理负担

**对比**: 商业 KVM 软件（如 Synergy/Barrier）会对鼠标移动做节流，通常限制在 60-100Hz。

---

### 2.8 【中等】切换协议缺少 ACK 机制

**文件**: `common/protocol.py`, `server/share_server.py`

```python
MSG_TYPE_SWITCH_ACK = 0x07   # 定义了但从未使用
```

- 服务端发送 `MSG_TYPE_SWITCH` 后立即切换状态到 REMOTE
- 客户端可能还没收到/处理切换命令，服务端就已经开始丢弃本地鼠标事件
- 没有确认机制意味着状态可能不一致

---

### 2.9 【中等】切换回本地的边缘检测在客户端和服务端双重执行

**文件**: `client/share_client.py` 第210-245行, `server/share_server.py` 第265-276行

**客户端** `_check_edge_for_return`:
- 在每次鼠标移动时检查是否到达远程屏幕边缘
- 有 500ms 的冷却时间

**服务端** `ScreenEdge` + `SwitchController`:
- 也在做同样的边缘检测
- 两者可能冲突或产生重复切换

**问题**: 服务端在 REMOTE 状态下不再监控本地鼠标（`_on_mouse_event` 直接 return），但 ScreenEdge 轮询线程仍在运行，检测的是本地鼠标位置而非远程鼠标位置。

---

### 2.10 【中等】服务器端接收数据缓冲区过小

**文件**: `network/tcp_server.py`, `network/tcp_client.py`

```python
data = client_socket.recv(1024)   # 服务器端
data = self.socket.recv(1024)     # 客户端
```

- 1024 字节的缓冲区在高频鼠标移动时可能不够
- EventDispatcher 的粘包处理虽然能正确路由，但小缓冲区导致频繁系统调用

**文件**: `config_layout.json`

```json

```

这个值既用于本地屏幕（服务端边缘检测），又可能用于远程屏幕（客户端返回检测），但两个屏幕的物理尺寸不同，10像素在 2560 宽屏幕和 1920 宽屏幕上的触发区域比例不一致。

## 3. 修改方案

### 3.1 核心架构改动：增量坐标模式

**目标**: 让鼠标在不同电脑间移动像在同一台电脑的多屏上一样丝滑。

**原理**: 物理多屏之所以丝滑，是因为：
1. 只有一个光标
2. 光标穿过边界时 Y 坐标按比例映射，X 坐标从边缘进入
3. 移动使用增量（delta），没有绝对坐标跳跃
4. 光标在源屏幕消失和目标屏幕出现是同步的

**实现方案**:

#### 3.1.1 新增增量移动消息类型

**文件**: `common/protocol.py`

```python
MSG_TYPE_MOUSE_DELTA = 0x08   # 增量鼠标移动
# 格式: [type(1B)] [dx(2B, signed)] [dy(2B, signed)]
# 总共 5 字节，比绝对坐标的 9 字节更小，且不需要持久的绝对坐标同步

def pack_mouse_delta(dx, dy):
    """打包鼠标增量移动事件"""
    return struct.pack(">Bhh", MSG_TYPE_MOUSE_DELTA, dx, dy)

def unpack_mouse_delta(data):
    """解包鼠标增量移动事件"""
    _, dx, dy = struct.unpack(">Bhh", data)
    return dx, dy
```

**优势**:
- 5 字节/事件 vs 9 字节/事件，减少 44% 带宽
- 天然支持不同分辨率（增量不受分辨率影响）
- 不需要绝对坐标同步

#### 3.1.2 服务端在远程模式下发送增量而非丢弃

**文件**: `server/share_server.py`

```python
def _on_mouse_event(self, data):
    if self.switch_controller and self.switch_controller.is_remote_control():
        # 不再 return，而是发送增量数据
        msg_type = get_msg_type(data)
        if msg_type == MSG_TYPE_MOUSE_MOVE:
            x, y = unpack_mouse_move(data)
            dx = x - self._prev_x
            dy = y - self._prev_y
            self._prev_x = x
            self._prev_y = y
            # 节流：忽略极小移动
            if abs(dx) < 1 and abs(dy) < 1:
                return
            delta_data = pack_mouse_delta(dx, dy)
            self._send_to_client(delta_data)
        return
    else:
        # 正常模式：记录位置用于未来增量计算
        msg_type = get_msg_type(data)
        if msg_type == MSG_TYPE_MOUSE_MOVE:
            x, y = unpack_mouse_move(data)
            self._prev_x = x
            self._prev_y = y
        if self._client_socket:
            try:
                self._client_socket.sendall(data)
            except Exception as e:
                print(f"[ShareServer] 发送鼠标事件失败: {e}")
```

#### 3.1.3 客户端支持增量模式移动

**文件**: `client/share_client.py`

```python
def _on_mouse_delta(self, dx, dy):
    """处理增量鼠标移动"""
    # 缩放增量以适配不同分辨率
    scale_x = self.local_screen.width / self._remote_screen_width
    scale_y = self.local_screen.height / self._remote_screen_height
    scaled_dx = int(dx * scale_x)
    scaled_dy = int(dy * scale_y)
    current = self.mouse_controller.get_position()
    new_x = current[0] + scaled_dx
    new_y = current[1] + scaled_dy
    self.mouse_controller.move_to(new_x, new_y)
```

### 3.3 重写插值为直接移动 + 预测

**文件**: `client/share_client.py`

**删除**当前的 `_interpolation_loop`，改用 `MousePredictor` 预测 + 直接移动：

```python
def _on_mouse_move(self, x, y):
    """处理绝对坐标鼠标移动（切换后的首次定位）"""
    self.target_x = x
    self.target_y = y
    if not self._first_move_received:
        self._first_move_received = True
        self.current_x = x
        self.current_y = y
        self.predictor.update(x, y)
        self.mouse_controller.move_to(x, y)
    else:
        self._check_edge_for_return(x, y)
        # 直接移动到目标位置，不做插值
        self.mouse_controller.move_to(x, y)
        self.current_x = x
        self.current_y = y
        self.predictor.update(x, y)

def _on_mouse_delta(self, dx, dy):
    """处理增量鼠标移动（远程控制期间）"""
    current = self.mouse_controller.get_position()
    new_x = current[0] + dx
    new_y = current[1] + dy
    self.mouse_controller.move_to(new_x, new_y)
    self._check_edge_for_return(new_x, new_y)
```

**不需要插值的原因**:
- 绝对坐标模式：增加节流后，帧率足够，直接移动比插值更跟手
- 增量模式：天然平滑，每步只是微小的偏移
- 插值引入人为延迟，反而降低体验

### 3.4 边缘检测改为事件驱动

**文件**: `server/screen_edge.py`

**删除轮询模式**，改为由 MouseListener 的事件驱动：

```python
class ScreenEdge:
    def __init__(self, on_edge_hit=None, threshold=EDGE_THRESHOLD):
        self.on_edge_hit = on_edge_hit
        self.threshold = threshold
        self._screen_width = 0
        self._screen_height = 0
        self._last_direction = None
        self._hit_count = 0
        self._hit_threshold = 2     # 连续2次在边缘才触发
        self._last_edge_time = 0
        self._cooldown = 0.2        # 200ms冷却

    def update(self, x, y):
        """由 MouseListener 事件调用的更新方法"""
        self._check_edges(x, y)

    def _check_edges(self, x, y):
        current_time = time.time()
        # 冷却检查
        if current_time - self._last_edge_time < self._cooldown:
            return

        direction = None
        if x <= self.threshold:
            direction = 'left'
        elif x >= self._screen_width - self.threshold:
            direction = 'right'
        elif y <= self.threshold:
            direction = 'top'
        elif y >= self._screen_height - self.threshold:
            direction = 'bottom'

        if direction and self.on_edge_hit:
            self._last_edge_time = current_time
            self.on_edge_hit(direction)
```

**改造 ShareServer**:
```python
# 在 MouseListener 回调中同时通知 ScreenEdge
def _on_mouse_event(self, data):
    msg_type = get_msg_type(data)
    if msg_type == MSG_TYPE_MOUSE_MOVE:
        x, y = unpack_mouse_move(data)
        # 通知 ScreenEdge（事件驱动，不轮询）
        if self._screen_edge:
            self._screen_edge.update(x, y)
    # ... 现有逻辑
```

### 3.5 修复坐标映射

**文件**: `server/screen_layout/coordinate_mapper.py`

**核心原则**: 鼠标穿过屏幕边界时：
- **进入方向的坐标应该固定为边缘值**（0 或 width-1）
- **垂直方向的坐标按比例映射**
- 不应该将 X 坐标也做比例映射（那是错的）

#### 修正后的右方向映射 (从本地右边缘到远程左边缘)

```python
@staticmethod
def _map_from_left(from_screen, to_screen, x, y):
    """
    鼠标从本地屏幕向右移动，到达右边缘后出现在远程屏幕的左边缘。
    
    物理效果：光标从本地右边缘移出，在远程屏幕左边缘同样高度位置出现。
    """
    # X: 进入远程屏幕左边缘
    new_x = 0
    
    # Y: 按比例映射
    new_y = int(y * to_screen.height / from_screen.height)
    
    # 根据 alignment 调整 Y 起始偏移
    if to_screen.alignment == 'top':
        new_y = int(y * to_screen.height / from_screen.height)
    elif to_screen.alignment == 'bottom':
        height_diff = to_screen.height - from_screen.height
        new_y = int(y * to_screen.height / from_screen.height) + max(0, height_diff)
        new_y = max(0, min(new_y, to_screen.height - 1))
    elif to_screen.alignment == 'center':
        y_offset = (to_screen.height - int(from_screen.height * to_screen.height / from_screen.height)) // 2
        new_y = int(y * to_screen.height / from_screen.height) + y_offset
    
    return (new_x + to_screen.offset_x, new_y + to_screen.offset_y)
```

#### 修正后的左方向映射 (从本地左边缘到远程右边缘)


### 3.7 添加鼠标事件节流

**文件**: `server/mouse_listener.py`

```python
import time

class MouseListener:
    MOUSE_MOVE_THROTTLE = 1/120     # 限制为120Hz
    MOUSE_MOVE_MIN_DELTA = 2        # 忽略2像素以下的移动

    def __init__(self, on_mouse_event=None):
        self.listener = None
        self.on_mouse_event = on_mouse_event
        self._last_x = 0
        self._last_y = 0
        self._last_move_time = 0

    def _on_move(self, x, y):
        now = time.time()
        # 节流：限制发送频率
        if now - self._last_move_time < self.MOUSE_MOVE_THROTTLE:
            return
        # 去抖：忽略极小移动
        if abs(x - self._last_x) < self.MOUSE_MOVE_MIN_DELTA and \
           abs(y - self._last_y) < self.MOUSE_MOVE_MIN_DELTA:
            return

        self._last_x = x
        self._last_y = y
        self._last_move_time = now
        if self.on_mouse_event:
            data = pack_mouse_move(x, y)
            self.on_mouse_event(data)
```

**节流频率选择**:
- 120Hz：足够流畅（8.3ms/帧），比大多数显示器刷新率高
- 远低于鼠标回报率（通常 1000Hz），减少网络负担约 8 倍

### 3.8 实现 ACK 切换确认机制

**文件**: `common/protocol.py`, `server/share_server.py`, `client/share_client.py`

```python
# protocol.py
MSG_TYPE_SWITCH_ACK = 0x07

def pack_switch_ack(switch_id):
    """打包切换确认"""
    return struct.pack(">BI", MSG_TYPE_SWITCH_ACK, switch_id)

def unpack_switch_ack(data):
    """解包切换确认"""
    _, switch_id = struct.unpack(">BI", data)
    return switch_id
```

```python
# server/share_server.py
def _on_switch_to_remote(self, target_screen, mapped_x, mapped_y):
    self._switch_id += 1
    self._pending_switch_id = self._switch_id
    switch_data = pack_switch(mapped_x, mapped_y, 0, self._switch_id)
    self._client_socket.sendall(switch_data)
    # 等待 ACK，超时 200ms 后也切换（容错）
    # 切换后隐藏光标

def _on_switch_ack(self, switch_id):
    if switch_id == self._pending_switch_id:
        self._apply_remote_mode()
```

```python
# client/share_client.py
def _on_switch(self, target_x, target_y, action, switch_id):
    # 立即移动鼠标到目标位置
    self.mouse_controller.move_to(target_x, target_y)
```

## 5. 建议的实施阶段

### 阶段一: 快速修复 (P0 项)
修复坐标映射、隐藏本地光标、开启 TCP_NODELAY。这3项改动量小，效果最明显。

### 阶段二: 核心优化 (P1 项)
将边缘检测改为事件驱动、重写插值逻辑为直接移动、添加鼠标节流。这3项解决切换响应延迟和移动卡顿。

### 阶段三: 增量模式 (P1 大项)
实现增量坐标传输模式，从根本上让鼠标移动像物理多屏一样丝滑。这是最终目标。

### 阶段四: 完善 (P2 项)
ACK 机制、缓冲区优化、返回冷却等锦上添花。

## 6. 其他改进建议

### 6.1 统一鼠标移动服务端处理流程

当前服务端有两套边缘检测机制在并行运行:
1. `ScreenEdge`（轮询模式，50ms间隔）
2. `SwitchController` + `EdgeDetector`（事件回调）

建议**移除 ScreenEdge 的轮询机制**，统一使用 SwitchController 的事件驱动方式。在 `ShareServer._on_mouse_event` 中直接调用 `SwitchController.on_mouse_move`。
